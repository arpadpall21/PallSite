System Flags (file descriptor permission)
Flag Data manipulation Description
a append open file for appending data to its content (creates a new file if it does not exist)
ax append creates a new file and appends the passed datas to it (fails if the file exist)
a+ read and append open file for reading and appending datas to its content (creates a new file if it does not exist)
ax read and append creates a new file for reading and appending datas to it (fails if the file exist)
as append (synchronous) open file for appending data to its content (creates a new file if it does not exist)
as+ read and append (synchronous) open file for reading and appending datas to its content (creates a new file if it does not exist)
r read open existing file for reading (fails if the file does not exist)
r+ read and write open existing file for reading and writing (fails if the file does not exist)
rs+ read and write (synchronous) open existing file for reading and writing (fails if the file does not exist)
w write open file for writing (existing file will be truncated, a new file will be crated if it does not exist)
wx write creates a new file for writing (file content will be truncated, fails if the file exist)
w+ read and write open file for reading and writing (existing file will be truncated, a new file will be crated if it does not exist)
wx+ read and write creates a new file for reading and writing (file content will be truncated, fails if the file exist)

Description and Demonstration :

- the Node.js file system API let us interact with files, the fs is closely modeled around standard POSIX functions (which is not helpful for me because I'm testing this on windows [TESTED : 2019-09-23])

- string file paths are interpreted as UTF-8 character sequences and resolved as absolute paths, (WHATWG) URL object as file paths must have fileURLs

- all fs methods are synchronous and asynchronous as well

- exceptions are immediately thrown on synchronous operations, which can be handled by using the try...catch statement

- exceptions on asynchronous operations are returned by the method's callback function as first argument (Node.js error first policy) 

// asynchronous exception handling ----------------------------------------------------
    fs.unlink('nonExistingFile', (err) => {
        console.log( err );                                                            // -> thrown exception returned here       
    });
    
// synchronous exception handling -----------------------------------------------------
    try { 
        fs.unlinkSync('nonExistingFile');                                              // if exception thrown the 'try' statement handles it (just like any other JS synchronous operations)   
    } catch(err) {
        err;                                                                           // -> thrown exception returned here 
    }

- the order is not garanteed when using asynchronous methods (this is normal asynchronous behaviour) so carefuly plan using asynchronous operations 

// asynchronous racing conditions -----------------------------------------------------
    fs.rename('oldName', 'newName', (err) => console.log(err) );                       // -! asynchronous rename method is NOT garanteed to be executed first 
    
    fs.readdir('./', function(err, files){                                             // -! asynchronous read directory method is NOT garanteed to be executed second 
        files;                                                                         // -> ['oldName', ...]         // !! we may have misleading informations here because of the asynchronous behaviour     
    })
    
// way to use asynchronous operations -------------------------------------------------
    fs.rename('oldName', 'newName', function(err){                                     // this way the call order is garanteed 
        if (err) throw err; 
        fs.readdir('./', function(err, files){
            console.log( files );                                                      // -> ['newName', ...]         // reliable informations 
        });
    });
