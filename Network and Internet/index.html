OSI model
Layer Description
7 - Application interface on which the user interacts, network application used by the user (Web Browser, Outlook, Skype, ect..)
6 - Presentation data translator (SSL, data translation, compression, encryption/decryption, ect...)
5 - Session open/closes and manages a communication session (link) with another device on the network (IP, ect..)
4 - Transport manages the actual data transport (TCP, UDP, ect...)
3 - Network path determination (data routing, logical addressing, ect...)
2 - Data Link transfering data between adjacent network nodes (Physical addressing, MAC, LLC, ect...)
1 - Hardware hardware itself (Network adapter, cable, ect...)

IP addresses
Address Description
IPv4
[0-255].[0-255].[0-255].[0-255] IPv4 address represented by 4 octets
10.0.0.0 10.255.255.255 (Class A) private IP range (16,777,216 addresses available)
172.16.0.0 172.31.255.255 (Class B) private IP range (1,048,576 addresses available)
192.162.0.0 192.162.255.255 (Class C) private IP range (65,536 addresses available)
127.0.0.0 127.255.255.255 - Loopback IP range (127.0.0.1 = localhost)
- network adapter self test (packets never sent through the network)
169.254.0.0 169.254.255.255 - Link-Local IP range = APIPA (Automatic Private IP Addressing)
- host randomly self assigns one in case the DHCP is not reachable
0.0.0.0 - means all ip addresses on this machine
IPv6
4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f] IPv6 represented by 8 blocks of 4 hextets
2|3ggg:gggg:gggg:ssss:iiii:iiii:iiii:iiii - Global Unicast is a public IPv6 address routable on the internet
- 2|3 - prefix (3bits)
- g - global routing (45bits) = globally assigned by IANA (global hierarchical assignement)
- s - subnet id (16bits) = identifies the subnet, assigned by the Administrator
- i - interface id (64bits) = individual node addresses on the subnet, assigned by the Administrator
fdgg:gggg:gggg:ssss:iiii:iiii:iiii:iiii - Unique Local is a private IPv6 address used in private networking (not routable on the internet)
- fd - prefix (8bits)
- g - global id (40bits) = assigned by the Administrator
- s - subnet id (16bits) = identifies the subnet, assigned by the Administrator
- i - interface id (64bits) = indivudual node addresses on the subnet, assigned by the Administrator
fe80:0000:0000:0000:iiii:iiii:iiii:iiii%id - Link Local used to communicate with neighbouring nodes (IPv6 nodes are automatically assigned by the host with this address)
- fe80 - prefix (10bits)
- i - interface id (64bits) = device id
- %id - scope id (on windows)
fffs:gggg:gggg:gggg:gggg:gggg:gggg:gggg - Multicast Address
- ff - prefix (8bits)
- f - flag (4bits)
- s - scope (4bits)
- g - group id (112bits)
::1 - loopback address, network adapter self test (packets never sent through the network)
:: - means all ip addresses on this machine

DNS Reccords
Record Description
TTL (Time To Live) defines how long (seconds usually) the DNS record should be cached by the requested host (fully explained

- when a machine requests the DNS a domain name resolution the record's TTL tells the requested machine for how long it should keep the DNS record cached

- let's say the www.someSite.com domain has a 1440 TTL (expressed in seconds which is 1 day)

- in this case the requested machine caches the www.someSite.com for one day, so it won't make any new DNS request for this domain name for one day

- once the record's TTL is expired the machine will make a new DNS request for this domain name again

)
A reccords - (Host IPv4) specifies domains or subdomains pointing to the set server(s)
- blank or @ = base/naked domain | * = all unspecified subdomain
AAAA records - (Host IPv6) specifies domains or subdomains pointing to the set server(s)
- blank or @ = base/naked domain | * = all unspecified subdomain
CName records (Canonical name) points to a domain or subdomain to another hostname
MX reccords - (Mail Exchanger) specifies which email server(s) to use when a mail is sent to the specified domain
- server priority can be set, the lower value has higher prioritiy (ex: 0 = highest priority)
NS records (authoritative Name Server) authoritative DNS server(s) where these DNS records are stored
TXT reccords (Text string) simple text records associated with the domain
PTR records ((domain name) Pointer) resolves an IP address to a fully-qualified domain name (does the oposite as A records) [NOT TESTED: 2020-02-25]
SRV records (Service) points service servers (like MX records but for other services) [NOT TESTED: 2020-02-25]
SOA records - (Start Of Authority) record appears at the beginning of the DNS zone
- contains the Authoritative Name Server, contact details for the domain Administrator and the domain serial number
CAA records - (Certification Authority Authorization) specifies which Certificate Authorities are allowed to issue certificates for a domain
- if nothing is specified all Authorities are allowed to issue certificats
NAPTR (Name Autority Pointer) used in internet telephony for mapping servers to users

The OSI model (Open System Interconnection)

- the OSI model is a conceptual model which characterizes and standardizes how telecommunication systems should work

- defines 7 conceptual layers, these layers do not refer to a specific structure or technology but rather they describe a function at the specified level

- the layer order defines the steps how the user sends or receives data through the network, on top (Application layer) is the user side and on the bottom (Hardware layer) is the hardware side
