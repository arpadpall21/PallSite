Description and Demonstration :

- if an application (web page) requests a resource from other than its own origin (the server which hosts this site) than it makes a cross-origin request

- http cross-origin requests are regulated by the CORS mechanism which uses http headers to control access cross-origin resources

- web servers include in their responses these headers (ex: Access-Control-Allow-Origin) which regulate the access if the resource was requested from an external (third party) site

- browsers' XMLHttpRequest and Fetch APIs can only request resources from the site's own origin (same-origin-policy) (unless the response from other origins include the right CORS headers)
