Address Description
IPv4
[0-255].[0-255].[0-255].[0-255] IPv4 address represented by 4 octets
10.0.0.0 10.255.255.255 (Class A) private IP range (16,777,216 addresses available)
172.16.0.0 172.31.255.255 (Class B) private IP range (1,048,576 addresses available)

- network adapter self test (packets never sent through the network)
169.254.0.0 169.254.255.255 - Link-Local IP range = APIPA (Automatic Private IP Addressing)
- host randomly self assigns one in case the DHCP is not reachable
IPv6
4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f] IPv6 represented by 8 blocks of 4 hextets
2|3ggg:gggg:gggg:ssss:iiii:iiii:iiii:iiii - Global Unicast is a public IPv6 address routable on the internet
- 2|3 - prefix (3bits)
- g - global routing (45bits) = globally assigned by IANA (global hierarchical assignement)
- s - subnet id (16bits) = identifies the subnet, assigned by the Administrator
- i - interface id (64bits) = individual node addresses on the subnet, assigned by the Administrator
fdgg:gggg:gggg:ssss:iiii:iiii:iiii:iiii - Unique Local is a private IPv6 address used in private networking (not routable on the internet)
- fd - prefix (8bits)
- g - global id (40bits) = assigned by the Administrator
- s - subnet id (16bits) = identifies the subnet, assigned by the Administrator
- i - interface id (64bits) = indivudual node addresses on the subnet, assigned by the Administrator
fe80:0000:0000:0000:iiii:iiii:iiii:iiii%id - Link Local used to communicate with neighbouring nodes (IPv6 nodes are automatically assigned by the host with this address)
- fe80 - prefix (10bits)
- i - interface id (64bits) = device id
- %id - scope id (on windows)
fffs:gggg:gggg:gggg:gggg:gggg:gggg:gggg - Multicast address
- ff - prefix (8bits)
- f - flag (4bits)
- s - scope (4bits)
- g - group id (112bits)
0000:0000:0000:0000:0000:0000:0000:0001 Loopback Address, network adapter self test (packets never sent through the network)

Description and Demonstration :

- an IP address (Internet Protocol) is a digital adderss on the network dynamically assigned to the device (required for TCP/IP network protocols)

- the authority what decides global (public) IP address distribution is called IANA (Internet Assigned Numbers Authority)

- a MAC address (Media Access Control) is a physical address staticaly assigned (unchangeable) to a network hardware

IP Address Types

- the IP address type defines how the data is routed to the destination

IP Type Description IPv4 IPv6
Unicast data sent to one specified destination YES YES
Anycast data is sent to the closest destination only (multiple destinations are specified) NO YES
Multicast data is sent to multiple specified destinataions YES YES
Broadcast data is sent to all nodes on the network YES NO

IPv4

- an IPv4 is made of 32bits divided into 4 blocks, each block is composed of 4bits (each block is represented by one octet (0-255)) 

        [0-255].[0-255].[0-255].[0-255]                                                // IPv4 address represented by 4 octets

IPv4 subneting

- IANA defined special IP addresses what can be used for private networks, private IP addresses are not routable on the internet (canot be used on the internet = routers prevent using them)

Subnet IP range Description
10.0.0.0 - 10.255.255.255 Class A (16,777,216 addresses available)
172.16.0.0 - 172.31.255.255 Class B (1,048,576 addresses available)
192.162.0.0 - 192.162.255.255 Class C (65,536 addresses available)

IPv4 Subnet Mask

- the subnet mask basically divides the IP addres into 2 parts, expressed in bits counted from left

- the left part identifies the network and the right part identifies the device

- this is where the sender device knows if the destination is on its network or on another network (sends to the gateway) 

        ipAddress/subnetMask                                                           // the subnetMask defines how many bits identifies the network (counted from left), the right part is the device identifier    
        
        [0-255].[0-255].[0-255].[0-255]                                                // subnet mask octet representation   
    
    10.1.1.102/8                                                                       // IP has an 8bit subnet mask (10 = network id) (1.1.102 = host id)
    255.0.0.0                                                                          // subnet mask   // underlying bits -> 11111111.00000000.00000000.00000000
    
// ------------------------------------------------------------------------------------
    10.1.1.102/16                                                                      // IP has a 16 bit subnet mask (10.1 = network id) (1.102 = host id)
    255.255.0.0                                                                        // subnet mask   // underlying bits -> 11111111.11111111.00000000.00000000
    
// ------------------------------------------------------------------------------------
    10.1.1.102/20                                                                      // IP has a 20 bit subnet mask (10.1.1 =network id) (1.102 host id)  // -! the 3rd octet (1) is halved    
    255.255.240.0                                                                      // subnet mask   // underlying bits -> 11111111.11111111.11110000.00000000
    
// ------------------------------------------------------------------------------------
    10.1.1.102/32                                                                      // IP has a 32bit subnet mask    // -! point to point link
    255.255.255.255                                                                    // subnet mask   // underlying bits -> 11111111.11111111.11111111.11111111
Example :

- our device IP address is 192.122.1.102/16

- if we send data to an address 192.122.1.104 the sender device will know that the destination is in our LAN because the target's subnet mask portion corresponds with the sender device

- if we sent data to an address 192.123.1.104 the sender device will know that the destination is not in our LAN because the target's subnet mask does not correspon with the sender device

IPv6

- an IPv6 is made of 128bits divided into 8 blocks, each block is composed of 16bits (each block is represented by 4 hextets (0-f))

- IPv6 is a different network, so we cannot send data to an IPv4 network from an IPv6 network (without transition technolgies like 6to4, ISATAP, Teredo) 

        4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]:4x[0-f]                // IPv6 represented by 8 blocks of 4 hextets  
        
    // IPv6 abbreviation rules --------------------------------------------------------
        48c7:0004:0000:a4d4:0000:0000:0c00:0001                                        
        48c7:4:0000:a4d4:0000:0000:c00:1                                               // 1) block leading 0's can be omitted 
        
        48c7:0004:0000:a4d4:0000:0000:0c00:0001
        48c7:0004:0:a4d4:0:0:0c00:0001                                                 // 2) block containing consecutive 0's abbreviated to one single 0   
        
        48c7:0004:0000:a4d4:0000:0000:0c00:0001
        48c7:0004:0000:a4d4::0c00:0001                                                 // 3) two or more blocks containing consecutive 0's are replaced to :: (abbreviation allowed only once!) (the longer 0 sequence is abbreviated if ther are several!)    
        
        48c7:0004:0000:a4d4:0000:0000:0c00:0001
        48c7:4:0:a4d4::c00:1                                                           // fully abbreviated IPv6

IPv6 address types

- IPv6 is hierarchically designed, multiple addresses are put into one large IPv6 (hierarchically), so IPv6 have no subnet mask (not in sence IPv4 has)

- IPv6 address types define what the address is used for, we can easily identify IPv6 types from their prefixes (first bits)

IPv6 Transition Technologies

- ISATAP (Intra-site Automatic Tunnel Addressing Protocol) router allows an IPv6 network to communicate with an IPv4 network (and vice versa) private networks only

- 6to4 allows the IPv6 traffic through an IPv4 network (cannot tunnel IPv4 traffic through IPv6 network)

- Teredo from an IPv4 network we connect to a Teredo Server/Relay which transform the traffic to IPv6
