Description and Demonstration :

- the data is broken into pieces and sent through the network by the sender device, then receiver device put back all together

- there are two main network protocols which control this proces, TCP and UDP

- data pieces travelling on the network are called Frames (on OSI layer 2) and Packet (on OSI layer 3)

NAT / PAT / Port Forwarding (Network Traffic Routing)

- IP (Internet Protocol) is a protocol which estabilishes the connection between to devices (operates on OSI layer 5)

- when we are talking about NAT we are talking about NAT and PAT because these two protocols are working together in real life

- NAT (Network Address Translation protocol) dynamically or statically translates Private IP addresses into Public IP addresses

- NAT by itself does a one to one translation, one private IP is translated to one public IP

- PAT (Port Address Translation protocol) is an extension of NAT, it translates port numbers

- PAT allows to use one public IP with multiple translated port numbers

- Port Forwarding (protocol) directs incoming traffic to a specified port (from network A) into a specific address on the other network (to network B)

TCP and UDP (transport protocols)

- TCP (Transmission Control Protocol) the receiver device confirms the received data packets, this makes sure that there is no data is lost over the network but it's slow

- UDP (User Datagram Protocol) the receiver device does not confirm the received data packets, this protocol is fast but there is a risk of data loss over the network
