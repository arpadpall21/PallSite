API Description
window|serviceWorker.fetch(url:str|req:obj

- if req (Request Object) is passed the 2nd argument (option:obj) overrides the Request Object's options!

, option:obj

method: str - request method (Default: 'GET')

headers: obj|headers:obj - request headers (object literal or headers object)

body: str|blob|bufferSource|formData|URLSearchParams|readableStream - request body

mode: str - CORS mode (Default: 'cors')

credentials: 'same-origin'|'omit'|'include' - credentials (cookies, authorization headers, TLS certificates) are sent with the request (Default: 'same-origin')

cache: str - request cache mode ('default' - if there's a match and its fresh the resource is returned from the cache) ('no-store' - direct request) ('reload' - direct request and reloads the browser cache) ('no-cache' - if there's a fresh or slate cached version the browser makes conditional request) ('force-chache' - fresh or slate version is returned from the cache, normal request otherwise) ('only-if-cached' - fresh or slate cached version only, no request made to the server)

redirect: 'follow'|'error'|'manual' ('follow' - automatically folow redirects) ('error' - aborts redirects with an error) ('manual' - manually handles redirects) (Default: 'follow')

referrer: str - rererrer of the request (same origin URL or 'about:client' or '' (empty string))

referrerPolicy: str - referrer policy for the request ('no-referrer' - the Referer header omited = no referrer information is sent with the request) ('no-referrer-when-downgrade' - if the provided value is invalid the origin path querystring of the URL are sent as referrer when the protocol security stays the same level or improuves but isn't sent when the protocol security downgrades (HTTPS -> HTTP) ) ('origin' - only the origin is sent as referrer (es: https://example.com/page.html -> https://example.com/) ('origin-when-cross-origin' - for cross-origin requests only the origin part is sent of the URL) ('same-origin' - for cross-origin request no referrer information is sent) ('strict-origin' - send the origin only when the security level stays the same (HTTPS -> HTTPS) ('strict-origin-when-corss-origin' - for cross-origin requests send the origin only when the security level stays the same (HTTPS -> HTTPS) ('unsafe-url' - send origin path querystring regardless of security)

integrity: str - contains the subresource integrity value (ex: sha256-BpfBw7ivV8q2jLiT13fxDYAe2tJllusRSZ273h2nFSE=) subresource integrity [link]

signal: abortSignal - abort signal to send in order to abort a pending request (ex: abortController.signal)

Headers guard none

- if the headers object is created by a Headers() constructor

- any request headers can be modified (custom headers included) in the headers object

Headers guard request

- if the headers object is created by a CORS req (Request) request

- any request headers can be modified (custom headers included) except forbidden request headers (silently discarded) in the headers object

Headers guard request-no-cors

- if the headers object is created by a NO-CORS req (Request) request

- only valid non-forbidden request headers can be set in the headers object (other headers are silently discarded)

Headers guard response

- if the headers object is created by a res (Response) object

- any response headers can be modified (custom headers included) except forbidden response headers (silently discarded) in the headers object

Headers guard immutable

- if the headers object is created by a res (Response) object which was created by a res.redirect() or a res.error()

- all headers are immutable in the headers object (TypeError is thrown if attempted to modify)

The AbortController and AbortSignal interfaces (abort request)

- the AbortController interfaces allows to abort one ore multiple Web requests

- the AbortSignal represents an abort signal which is sent to a DOM request (Web request) in order to abort it 

SYNTAX:     new AbortController()                                                      // creates and returns an abortController object which is used to abort Web requests (DOM requests (fetch))    
            
            
            abortController.signal                                                     // returns the abortSignal object of the abort controller (this is where the abort controller will send the abort signal)    
            abortController.abort()                                                    // sends the abort signal to the abortController.signal    
            
            
  // AbortSignal ----------------------------------------------------------------------
            abortSignal                                                                // returned by abortController.signal represents the abort signal itself of the abort controller    
            
            
            abortSignal.aborted                                                        // returns true if the abort signal is sent and the request is aborted, otherwise false    
            
            
            abortSignal.addEventListener('abort', fn(ev))                              // event triggered when signal is sent and the request is aborted (cannot be used as event property (abortSignal.abort = fn(ev)))
DEMO 
<script>
// AbortController (signal property) --------------------------------------------------
    var controller = new AbortController();                                            // creates and returns an abort controller object   
    var signal = controller.signal;                                                    // controller object's abort signal   
    
// AbortSignal (aborted property / aobrt event) ---------------------------------------
    signal.addEventListener('abort', function(ev){                                     // event emitted when the signal is sent and the request is aborted    // -! this event cannot be used as event property (signal.abort = function(ev){})    
        signal.aborted;                                                                // -> true           // signal successfully aborted the DOM request    
    });
    
    fetch('http://localhost:1000', {signal})                                           // abort signal is passed (but not sent) as fetch option (ES6 syntax same as {signal:signal})    
        .then(res=> {
            res.text()
                .then(data=>console.log( data ) )
                .catch(err=>console.log(err));
        })
        .catch(err=>console.log(err));                                                 // fetch promise rejected (because the request is aborted)    
        
// AbortController (abort() method) ---------------------------------------------------
    controller.abort();                                                                // abort controller sends its abort signal   
</script>

The Headers interface

- the Headers interface allows us to perform various actions on HTTP request and response headers 

SYNTAX:     new Headers(obj|headers

                
 - obj - object literal 

                
 - headers - another headers object 

            
)                                                  // creates and returns a new headers object (iterable object for... of loop can be used on it)
                                                                                          header names and values accept ASCII characters only, header names are (parsed to) lowercased    
                                                                                          implements header guard

                
 Headers guard none 

                
 - if the headers object is created by a Headers() constructor 

                
 - any request headers can be modified (custom headers included) in the headers object 

                    

                
 Headers guard request 

                
 - if the headers object is created by a CORS req (Request) request 

                
 - any request headers can be modified (custom headers included) except forbidden request headers (silently discarded) in the headers object 

                    

                
 Headers guard request-no-cors 

                
 - if the headers object is created by a NO-CORS req (Request) request 

                
 - only valid non-forbidden request headers can be set in the headers object (other headers are silently discarded) 
 
                    

                
 Headers guard response 

                
 - if the headers object is created by a res (Response) object 

                
 - any response headers can be modified (custom headers included) except forbidden response headers (silently discarded) in the headers object 

                    

                
 Headers guard immutable 

                
 - if the headers object is created by a res (Response) object which was created by  a res.redirect() or a res.error() 

                
 - all headers are immutable in the headers object (TypeError is thrown if attempted to modify) 

            
 mechanism that controlls what headers can be accessed depending how the headers object was created    
            
            
            headers.set(headerName:str, val)                                           // sets a new or overwrites an existing header   
            headers.append(headerName:str, val)                                        // sets a new or appends a new value to an existing header   
            
            
            headers.delete(headerName:str)                                             // deletes the specified header from the headers object   
            
            
            headers.get(headerName:str)                                                // returns the value of the specified header name    
            headers.has(headerName:str)                                                // returns true if the specified header name exists in the headers object, false otherwise    
            
            
            headers.entries()                                                          // returns an iterator object which contains the headers key/value pairs   
            headers.keys()                                                             // returns an iterator object which contains the headers keys    
            headers.values()                                                           // returns an iterator object which contains the headers values    
            
            
            headers.forEach(fn(val, key, headers))                                     // runs the callback for each key/value pairs in the headers object
