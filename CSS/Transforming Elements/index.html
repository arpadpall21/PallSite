Remember This :

- HTML elements cannot have any thickness

- CSS transformations do not stack (like in canvas), if you write a new CSS transform line it will completely overrides the previous transformation! (this is a normal CSS behaviour)

- in the browser the perspective is counted from the viewer towards the plain z=0, so the lower is the perspective value the bigger is the space between the user and the plain z=0 (greater is the 3D effect)

- the translateZ() function and moving the Z origin point with the transform-origin property does the same exact thing with different approaches (both moves the element along the Z axis but the Z origin point is always kept in the same place (on the plane))

Description and Demonstration :

- CSS lets us transform block level elements in 2D or 3D such as rotate, skew, ect..

- HTML elements cannot have any thickness

- each HTML element have their own transformation space, elements (with their unique space) are placed on top of each other flat hierarchically (think of like a stack of tiles)

- CSS transformations do not stack (like in canvas), if you write a new CSS transform line it will completely overrides the previous transformation! (this is a normal CSS behaviour)

the transfrom property 

SYNTAX :    transform : none                    // (default)            // transforms an element in 2D or 3D (transformed element does not affect the layout)                             
                      : translate(xCssLength, yCssLength)  // translates (moves) the element along the X and Y axis
                      : translateX(CssLength)   // translates (moves) the element along the X axis
                      : translateY(CssLength)   // translates (moves) the element along the Y axis
                      : translateZ(CssLength)   // translates (moves) the element along the Z axis (between the user and the page, lower value moves the element away from the viewer)
                      : translate3d(xCssLength, yCssLength, zCssLength)   // translates (moves) the element along the X, Y and Z axis
                      : scale(x:nr, y:nr)       // scales (sizes) the element along the X and Y axis
                      : scaleX(nr)              // scales (sizes) the element along the X axis (default 1)
                      : scaleY(nr)              // scales (sizes) the element along the Y axis (default 1)
                      : scaleZ(nr)              // scales (sizes) the element along the Z axis (between the user and the page) (default 1) (since HTML element cannot have any thickness this is very situational!)   
                      : scale3d(x:nr, y:nr, z:nr) // scales (sizes) the element along the X, Y and Z axis (default 1)
                      : skew(x:nrdeg, y:nrdeg)  // skews the element along the X and Y axis
                      : skewX(nrdeg)            // skews the element along the X axis 
                      : skewY(nrdeg)            // skews the element along the Y axis 
                      : rotate(nrdeg)           // rotate the element along the Z axis (same as rotateZ()) 
                      : rotateX(nrdeg)          // rotate the element along the X axis 
                      : rotateY(nrdeg)          // rotate the element along the Y axis 
                      : rotateZ(nrdeg)          // rotate the element along the Z axis (same as rotate()) 
                      : rotate3d(nr, nr, nr, nrdeg) // rotate the element along the X, Y and Z axis (creates a custom rotation axis, [unnecessarily complicated to use TESTED : 2019-05-09]) 
                      : perspective(CssLength)  // sets the distance (creates a space) between the user and the page (plane z=0) (this makes 3D transformation visible, must be set before the actual transformation)   
                      : matrix(nr, nr, nr, nr, nr, nr)   // shorthand 2d transform (not intended to be written by hand [I think]) [NOT TESTED]
                      : matrix3d(nr, nr, nr, nr, nr, nr, nr, nr, nr, nr, nr, nr, nr, nr, nr, nr)  // shorthand 3d transform (not intended to be written by hand [I think]) [NOT TESTED]
                      : transformFn() transformFn() ...  // multiple transform 
    
// HTML --------------------------------------------
    <div class="testDiv"></div>
    
// CSS ---------------------------------------------
    .testDiv {
    // transform / perspective ------- 
        transform: translateX(250px);                                   // translates (moves) the element 250px left
        transform: translateY(-25px);                                   // translates (moves) the element 25px up       // !!  at this point the element is translated along the Y axis ONLY because CSS transform do not stack (this is normal CSS behaviour)   
        transform: perspective(500px) translateZ(-250px);               // translates (moves) the element away from the user 250px(towards the plain z=0)   // -! perspective must be set before the actual transformation     
        // ------
        transform: translate(250px, -25px);                             // does the same as the 2D transformation above 
        transform: perspective(500px) translate3d(250px, -25px, -250px);// does the same as the 3D transformation above 
    
    // scale -------------------------
        transform: scaleX(1.5);                                         // scales the element along the X axis (stretches one and a half of its original size)
        transform: scaleY(.75);                                         // scales the element along the Y axis (shrinks three fourth of its original size)
        transform: scaleZ(2);                                           // scales the element along the Z axis    // -! since in HTML elements does not have any thickness this is very situational (I don't think it worth working with)   
        // ------
        transform: scale(1.5, .75);                                     // does the same as the 2D scale above 
        transform: scale3d(1.5, 0.75, 2);                               // does the same as the 3D scale above 
        
    // skew --------------------------
        transform: skewX(55deg);                                        // skews the element along the X axis 
        transform: skewY(-25deg);                                       // skews the element along the Y axis 
        // ------
        transform: skew(55deg, -25deg);                                 // does the same as above 
    
    // rotate --------------------------
        transform: rotate(45deg);                                       // rotate the element along the Z axis    // -! same as "rotateZ()"
        transform: rotateZ(45deg);                                      // rotate the element along the Z axis 
        transform: rotateX(75deg);                                      // rotate the element along the X axis 
        transform: rotateY(35deg);                                      // rotate the element along the Y axis 
        transform: rotate3d(1, 1, 1, 45deg);                            // rotates the element along the X, Y and Z axes 45 degrees     // -! I've found this overcomplicated to use [TESTED: 2019-05-11]
    }

the transform-origin property 

SYNTAX :    transform-origin: x:cssLength y:cssLength z:cssLength   // (default 50% 50% 0)    //  determinates the central transformation point on which the element is transformed around ("hanging point") (when we set the "z:cssLength" value the Z origin point stays in the same place but the element is moved along the Z axis, in other words this translates the element along the Z axis (does the same as the translateZ() function))    
                - x:cssLength : x axis coordinates (also can be left | center (default) | right)
                - y:cssLength : y axis coordinates (also can be top | center (default) | bottom)
                - z:cssLength : z axis coordinates (keeps the Z origin point at the same place and moves (translates) the element along the Z axis) 
    
// HTML --------------------------------------------
    <div class="demo"></div>
        
// CSS ---------------------------------------------
    .demo {
        transform-origin: left top;                                     // you can think of as a "hanging point" which is moved to the top left corner here
        transform: scale(.75, .75) rotateZ(-25deg);                     // the scale and the rotation now is happening around the top left corner 
    }
    
  // transform-origin Z parameter ----------------
    .demo {
        transform-origin: 50% 50% -100px;                               // the Z origin point is moved 100px behind the element  // -! by doing this it acually keeps the Z origin point in the same place and moves the element forward to the viewer   
        transform: perspective(500px);
    }
    .demo {
        transform: perspective(500px) translateZ(100px);                // this does the same thing az the code above (moves the element forward to the viewer and leaves the Z origin point behind)  
    }

the perspective / perspective-origin properties 

SYNTAX :    perspective: none                   // (default)            //  defines the perspective depth for the element's 1st level children elements (1st level children elements will share and transformend on the same space) 
                       : cssLength              // lower value means bigger perspective space 
            
            perspective-origin: x:cssLength y:cssLength   // (default 50% 50%)  //  defines at which angle the veiwer is looking the 3D transformed element (think as a "vanishing point" used by the perspective property) (effects chidren elements not the element itself)   
                - x:cssLength : also can be left | center (default) | right
                - y:cssLength : also can be top | center (default) | bottom
    
// HTML --------------------------------------------
    <div class="demo">
        <div>1</div>
        <div>2</div>
    </div>
    
// CSS ---------------------------------------------
    .demo {
        perspective: 200px;                                             // perspecitve is set for all children elements     // -! does not set any perspective for the parent itself!
        perspective-origin: left top;                                   // children elements in perspective are shown from the top left angle  
    }
    .demo > div:first-child {
        transform-origin: left;
        transform: rotateY(45deg);                                      // we could achive the same result here by setting the "perspective()" function for the transform property  
    }
    .demo > div:last-child {
        transform-origin: right;
        transform: rotateY(-45deg);
    }

the transform-style / backface-visibility properties 

SYNTAX :    transform-style:                                            //  defines if the element's 1st level children should be rendered in the same space as the current element [Experimental : 2019-05-13]   
                           : flat               // (default) 1st level children elements keep their own space and rendered flat in front of the current element
                           : preserve-3d        // 1st level children elements will be rendered in the space as the current element
        
            backface-visibility: visible        // (default)            //  defines if the backface of the element is visible when it's facing the viewer [Experimental : 2019-05-13]   
                               : hidden 
    
// HTML --------------------------------------------
    <div class="demo">
        <div></div>                                                     // child element is rendered in the same space as the parent because of the "transform-style:preserve-3d;" declaration   
    </div>
    
// CSS ---------------------------------------------
    .demo {
        transform: perspective(400px) rotateY(-25deg);
        transform-style: preserve-3d;                                   // 1st level child elements are rendered in the space of this element 
    }
    .demo div {
        transform: rotateY(25deg);                                      // because this element is rendered in the same place as its parent they sort of blend together 
        backface-visibility: hidden;                                    // the back side of this element will be hidden when it's facing the user
    }
