the display property 

SYNTAX :    display: block                                          // specifies the display behaviour (rendering behaviour) of the element 
                   : contents               // removes from the layout this element only (but not its children)
                   : flex                   // block-flex container
                   : grid
                   : inline 
                   : inline-block
                   : inline-flex            // inline-flex container
                   : inline-grid
                   : inline-table           // inline <table> element 
                   : list-item              // <li> element 
                   : none                   // removed from the layout (and all its children)
                   : table                  // <table> element 	
                   : table-caption          // <caption> element 	
                   : table-column-group     // <colgroup> element 	
                   : table-header-group     // <thead> element 
                   : table-footer-group     // <tfoot> element 	
                   : table-row-group        // <tbody> element 	
                   : table-cell             // <td> element 
                   : table-column           // <col> element 
                   : table-row              // <tr> element 
    
-- HTML -- 
    <p id="demo"> <u>children element</u> </p>
    <p id="demo2">demo paragraph <u>children element</u> demo paragraph</p>
    
-- CSS -- 
// block ---------------------------------------------------------------------- 
    #demo { display:inline; }                                       // block element display value changed to inline 
    
// none ----------------------------------------------------------------------- 
    #demo { display:none; }                                         // element and its children are removed from the layout
    
// contents ------------------------------------------------------------------- 
    #demo { display:contents; }                                     // only the element is removed from the layout but not its children 
                                                                    // -! displays : <u>children element</u>
    #demo2 { display:contents; }                                    // !! interestingly the targeted element's content is NOT removed eighter [might be a bug??? TESTED 2019-04-12]

the position property

- the position property makes possible to position elements on the page (without this property we cannot use any positioning properties like left / right / ect...)

position:value can be positioned affects layout positioned relative to
static NO YES -
relative YES YES its "static" position
fixed YES NO viewport
absolute YES NO its parent (parent element must be set to position:relative|absolute|fixed;)
sticky when "fixed" ONLY! YES - rendered with the flow until it reaches the offset position by scrolling
- viewport when the scroll tries to hide it away but limited within the parent element's boundaries

- the sticky element is positioned with the document's flow, it sticks on the viewport (switches to "position:fixed;") when the scrolling reaches the offset position (when the scroll tries to hide it away form the viewport) but it is limited inside its parent container (works only if overflow:visible;) 

// static --------------------------------------------------------------------- 
    .myElement { 
        position:static;                                            // default position rendered with the document flow 
        left: 10px;                                                 // -! this is not working (static element cannot be positioned)
        top: 10px;
    }
     
// relative ------------------------------------------------------------------- 
    .myElement { 
        position:relative;                                          // rendered with the document flow but can be positioned
        left: 10px;                                                 // positioned relative to its original place leaves a hole 
        top: 10px;
    }
    
// fixed ---------------------------------------------------------------------- 
    .myElement { 
        position:fixed;                                             // removed from the document's flow and placed permanently on the viewport 
        left: 10px;                                                 // always visible on the top left corner 
        top: 10px;
    }
    
// absolute ------------------------------------------------------------------- 
    .myElement { 
        position:absolute;                                          // removed from the document's flow     // -! parent element's position must be set to "relative"|"absolute"|"fixed"
        left: 10px;                                                 // relative to the parent element (floats inside the parent element) 
        top: 10px;
    }
    
// sticky --------------------------------------------------------------------- 
    .myElement { 
        position:sticky;                                            // rendered with the flow untill the scroll position tries to hide it away than it switches to "fixed", it stays visible on the viewport till it has room inside its parent element   
        top: 10px;                                                  // top treshold when the element hits this it will "stick" on the viewport IF it has room inside its parent 
        bottom: 10px;                                               // bottom treshold when the element hits this it will "stick" on the viewport IF it has room inside its parent 
    }                                                                   // in this example there is no horizontal treshold set, so horizontally the element will sroll away and never sticks 
                                                                    // -! we can set all 4 side tresholds to define where the sticky element should switch to fixed

the float and clear properties

- the "float" property does not work if position:absolute and the JavaScript property for it is style.cssFloat

SYNTAX :    float: none                                             // places the element left|right of its container, removes it from the layout relative to block-level elements, does NOT remove it from the layout relative to the text, inline elements and other floating elements in any level   
                 : left
                 : right 
                 
            clear: none                                             // specifies a side of a block-level element which does not allow floating elements (starts a new line) (applies on floating or non-floating elements)
                 : left
                 : right
                 : both                     // left and right 
    
-- HTML -- 
    <div class="demoFloat"></div>                                   // element floats on the left of the line 
    <div class="demoFloat"></div>                                   // element floats on the left of the previous floating element 
        <span> Inline Element </span>                               // inline element is NOT affected by any floating behaviour (so it just tries to follow the line)
    <div> text of the block-level element </div>                    // this block-level element is behind the two floating ones because floating elements are removed from the layout relative to block-level elements  
                                                                    // -! the block-level element's content : "text of the block-level element" surrounds the floating elements (becuase it is still an inline element)   
    <div id="demoBlock"> text of the block-level element  </div>    // this block-level element starts a new line relative to the floating elements because of the "clear" property
        
-- CSS -- 
    .demoFloat { float:left; }
    #demoBlock { clear:left; }                                      // element does not support any floating elements on its left side (will result in a new line)
