CSS Units

- in CSS we have 2 types of length units, absolute (fixed size) and relative (relative to another lenght)

Absolute CSS units

- using absolute units is not recommended on screen devices because the screen size can vary so much

Relative CSS units 

SYNTAX :    nrem                                                                       // relative to the font size of the element
            nrrem                                                                      // relative to the font-size of the root (<html>) element
            nrex                                                                       // relative to the "x-height" of the current font  (not reliable [TESTED 2019-01-14) 
            nrch                                                                       // relative to the widht of the 0(zero) character  (not reliable [TESTED 2019-01-14) 
            nrvw                                                                       // relative (percent) to the current width of the viewport
            nrvh                                                                       // relative (percent) to the current height of the viewport
            nrvmin                                                                     // relative (percent) to the smallest side (width or height which is currently the smallest) of the current viewport  
            nrvmax                                                                     // relative (percent) to the largest side (width or height which is currently the largest) of the current viewport
            nr%                                                                        // relative to the same property of the parent element
            nrpx (1px = 1/96th of 1in)                                                 // pixels (absolute pixel size but relative to the screen because of pixel density)
    
// em / rem ---------------
    .myElement {
        font-size: 2em;                                                                // double of the current font-size
        width: 10rem;                                                                  // 10 times bigger then the root element "<html>" font-size
    }
    
// % ----------------------
    .parentElement {
        height: 500px;
    }
    .childElement {
        height: 80%                                                                    // relative to the parent element's same property (500 * 80% = 400px)
    }
    
// vw / vh / vmin / vmax --
    .myElement {
        width: 10vw;                                                                   // 10% of the current viewport width
        height: 10vh;                                                                  // 10% of the current viewport height
        font-size: 2vmin;                                                              // 2% of the smallest side (width or height of the current viewport which is curently the smallest)  
        padding: 3vmax;                                                                // 3% of the largest side (width or height of the current viewport which is curently the largest)  
    }

The calc() function 

SYNTAX :    calc(cssLength +|-|*|/ cssLength ...)                                      //  performs a calculation of the passed values and returns the result
    
    .myElement {
        width: 25%;                                                                    // fallback for browsers not supporting calc() function
        width: calc(25% + 65px);                            
    }

The var() function CSS custom properties

- CSS custom properties are "sort-of" variables in CSS, they are used to insert property values (only!)

- CSS variables are scoped (available only on the element and on all its nested child elements) 

SYNTAX :    parentElement { --varId: value; }                                          // declares a variable and its value on the parent element (CSS varialbes are scoped, this variable is available only on nested elements)    
              
            childElement { property: var(--varId, fallbackValue); }                    // variable value can be used as property value only on child elements
                - fallbackValue : value used if the variable's value is undefined 
    
    <div>
      <div class="demo">
        <p> my element <span>child element</span></p>  
      </div>
    </div>
    
-- CSS --
    .demo {
      --myColor: red;                           // custom values declared in the parent element
      --myBorder: solid 2px blue;
      border: var(--myBorder);                  // custom value used on the element itself
    }
    .demo p {
      color: var(--myColor);                    // custom value used on the child element (custom properties inherit just like normal properties (the <span> child element inherit this color automatically) 
      border: var(--myBorder);                  // the <span> child element does NOT inherit this property because the "border" property non-inheritable 
      font-size: var(--myFontSize, 2em);        // fallback value is used because the "--myFontSize" identifier is undefined
    }
    div {
      border: var(--myBorder);                  // ancestors do not inherit custom properties (because of the scoping mechanism)
    }

@ Rules

The @charset / @import rule 

SYNTAX :    @charset "charset"                  // specifies the character encoding for the CSS file (must be the first declaration / HTTP header overrides it / cannot be placed as inline style or between the "<style>" tags in the head of the HTML document)   
        
            @import "URL" mediaQueryCondition   // imports a stylesheet into the current stylesheet (first declaration after the @charset if any)  
                     url("URL") mediaQueryCondition  
                - URL : references the css file (absolut or relative URL)  
    
// -- css document ---
    @charset "utf-8";                           // character encoding set   // -! first declaration in the css file 
    @inport url("other Stylesheet.css") screen and (min-width:800px);       // css file from another source imported to the current css file with a media query condition

The @supports rule and CSS.supports() JS property 

SYNTAX :    @supports not (prop:val) not|and|or (prop:val) ...{ }           // fires the code block if the passed property and value are supported by the browser (if the @supports at rule is not supported by the browser the entire code block is ignored)   
                - not : flips the value of the next property-value pair expression 
                - and : both expressions must be supported by the browser 
                - or : one of the exressions must be supported by the browser 
            
            window.CSS.supports(cssProp:str, cssVal:str)                    // returns a boolean value indicating if the browser supports the CSS property-value pair or the @supports conditions 
            window.CSS.supports(cssSupportsCondition:str)   
    
// (CSS) @supports rule ------------------------
    @supports(cursor:zoom-in) {                 // if the browser supports the 'cursor:zoom-in' property-value pair the code block is executed   
        .demo:hover { cursor:zoom-in } 
    }
    
    @supports ((not (newProp:newVal)) and (caret-color:initial)) or (text-decoration-line:underline) {      // multiple conditions and operators combined  
        .demo:hover { cursor:zoom-in }                                                                      // if all the above conditions are supported by the browser this code runs
    }
        
// (JavaScript) CSS.supports -------------------
    CSS.supports('cursor', 'zoom-in');                                                                      // -> true      // property and value pair supported by the browser 
    
    CSS.supports('((not (newProp:newVal)) and (caret-color:initial)) or (text-decoration-line:underline)')  // -> true      // @supports conditions are supported by the browser
