Remember This :

- an element cannot be smaller than its padding + border, the box-sizing:border-box manages the available space of the content part only

- because block-level elements take as much space as they can horizontally the min-width and max-width properties affected by that behaviour right away

Description and Demonstration :

Box Model and its dimension box-sizing

- All HTML elements considered as boxes, in CSS we call this "box-model"

- the "box-model" is composed of content / padding / border / margin

- when we set the size of an element by default we set the size of the content only (padding and border adds up to calculate the element's real size)

- the box-sizing property gives us the "power" to define which parts of the box-model are included when we calculate the elemnt's size

- an element cannot be smaller than its padding + border, the box-sizing:border-box manages the available space of the content part 

SYNTAX :    box-sizing: content-box         // (default) content                             //  defines what parts of the box-model include when we set the element's size
                      : border-box          // content + padding + border 
    
    <div class="demo d1"></div>
    <div class="demo d2"></div>
    
-- CSS --
    .demo {
        padding: 20px;
        border: solid black 30px;
        width: 200px;                       
        height: 100px;
    }
    
// -> d1 element's width : 200px + (2x20px) + (2x30px) = 300px
// -> d1 element's height : 100px + (2x20px) + (2x30px) = 200px
    .d1 {
        box-sizing: content-box;            // (default value) "width" and "height" properties sets the dimensions for the content area
    }
    
// -> d2 element's width: 200px
// -> d2 element's height : 100px
    .d2 {
        box-sizing: border-box;             // padding + border are included in the size (manages the space of the content)
    }

Sizing elements

width / min-width / max-width / height / min-height / max-height 

SYNTAX :    width: auto                     // (default) browser handles the width         // sets a fixed-width for the element
                 : ccsLength
                 : nr%                      // relative to the parent element 
                 
            min-width: ccsLength            // (default 0)                                 // defines the minimal width of an element (element can be wider if something forces it (like block-level behaviour or content))  
                     : nr%                  // relative to the parent element 

            max-width: ccsLength            // (default 0)                                 // defines the maximal width of an element (element can be smaller if nothing forces it (like block-level behaviour or content))  
                     : nr%                  // relative to the parent element 

            height: auto                    // (default) browser handles the height        // sets a fixed-height for the element
                  : ccsLength
                  : nr%                     // relative to the parent element 
                  
            min-height: ccsLength           // (default 0)                                 // defines the minimal height of an element (element can be higher if something forces it (like content))  
                     : nr%                  // relative to the parent element 

            max-height: ccsLength           // (default 0)                                 // defines the maximal height of an element (element can be shorter if nothing forces it (like content))  
                     : nr%                  // relative to the parent element       
    
    <p class="mins"> TEST </p>
    <p class="maxs"> TEST </p>
    
-- CSS  --
    .mins {
        background-color: darkorchid;
        min-width: 500px;                   // because this is a block-level element it takes up the full width of the screen but never gets narrower than 500px  
        min-height: 500px;                  // element height is 500px  but it would get higher automatically if we had a large content what forced it 
    }
    .maxs {
        background-color: darkorchid;
        max-width: 500px;                   // because this is a block-level element (it takes as much width as it can) this elemet is 500px wide but it gets narrower if the screen becomes narrower than 500px  
        max-height: 500px;                  // element height is 1em  but its height could be up to 500px if we would had a large content 
    }

explanation why the *-width properties work differently than the *-height properties on block level elements

- this block level elemet gets as wide as it can but gets as height as its content, this is the reason why the min-width takes the full width but the min-height takes only the necessary space for its content!

- BLOCK LEVEL ELEMENT
- min-width:1000px; (element takes the full available width because of the block level behaviour)
- min-height:100px; (element is 100px height, because the content does not forces it to get heigher)

- INLINE-BLOCK ELEMENT
- min-width:1000px; (element is 1000px wide because the inline-block element does take only the necessary width)
- min-height:100px; (element is 100px height, because the content does not forces it to get heigher)

resize 

SYNTAX :    resize: none                    // default                                     //  allows and defines how the user can resize the element (overflow:auto|scroll|hidden must be declared (no EDGE / IE support))    
                  : vertical
                  : horizontal
                  : both                    // vertical and horizontal 
    
    <div class="demo"> myContent </div>
    
-- CSS --
    .demo {
        width: 100px;
        height: 100px;
        border: solid 1px black;
        overflow: auto;                     < -- required in order to resize 
        resize: both;                       // element resizable both vertical and horizontally
    }

Placing Elements

top / right / bottom / left 

SYNTAX :    top: auto                       // (default) browser calculates the position   // sets the top edge position of the element ("position:static" = has no effect / "position:relative" = relative to its static position / "position:absolute" = relative to its parent / "position:fixed" = relative to the viewport)  
               : cssLength
               : nr%                        // relative to the parent element 
        
            right: auto                     // (default) browser calculates the position   // sets the right edge position of the element ("position:static" = has no effect / "position:relative" = relative to its static position / "position:absolute" = relative to its parent / "position:fixed" = relative to the viewport)  
                 : cssLength
                 : nr%                      // relative to the parent element 
                 
            bottom: auto                    // (default) browser calculates the position   // sets the bottom edge position of the element ("position:static" = has no effect / "position:relative" = relative to its static position / "position:absolute" = relative to its parent / "position:fixed" = relative to the viewport)  
                  : cssLength
                  : nr%                     // relative to the parent element 
                  
            left: auto                      // (default) browser calculates the position   // sets the left edge position of the element ("position:static" = has no effect / "position:relative" = relative to its static position / "position:absolute" = relative to its parent / "position:fixed" = relative to the viewport)  
                : cssLength
                : nr%                       // relative to the parent element       
    
    <p class="demo1"> static </p>
    <p class="demo2"> relative </p>
    <p class="demo2"> absolute </p>
    <p class="demo4"> fixed  </p>
-- CSS  --
    .demo1 {
        position: absolute;
        top: 100px;                         // placement has no effect because element is NOT poistioned (position:absolute)
    }
    .demo2 {
        position: relative;
        left: 100px;                        // placed 100px left to its "static" position
    }
    .demo3 {
        position: absolute; 
        right: 100px;                       // placed 100px right to its parent's right border (in this situation relative to the document (no parent element with "relative" position))
    }
    .demo4 {
        position: fixed;
        bottom: 100px;                      // placed 100px bottom relative the the viewport
    }
