Description and Demonstration :

- Semantic elements are elements with meaning (describles its meaing both for the browser and for the devloper (like : <form> / <table>)

The <header> / <nav> / <main> / <article> / <section> / <aside> / <footer> tags

- there is no rule or definition how these elements nest each other, it is up to the developer and the current page structure how we use them (like the "article" element can nest "section" elements or vice versa)

The <details> / <summary> / <figure> / <figcaption> tags

DEMO 
// <details> / <summary> -------------------
    <details>
        <summary> myDetailst title <summary>
         <CODE> 
    </details>
    
// <figure> / <figcaption> -----------------   
    <figure>
        <img src="myImage.jpg" alt="myImage">
        <figcaption> myImage caption </figcaption>
    </figure>
    
// <data> / <time> -------------------------    
    <p>It is <time datetime="16:31">sixteen hours 31</time> right now and we are on the <time datetime="2019-01-25">2019 25th of jaunary</time></p>
    <p> This color is <data value="red">red</data> </p>
    
// <mark> ----------------------------------------
    <p> This is a <mark>highlighted text</mark> </p>
