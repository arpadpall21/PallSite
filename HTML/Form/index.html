Form attributes accept-charset / action / method / enctype / name / target / autocomplete

- accept-charset attribute specifies the form character encoding (by default the form uses the document's encode) but this attribute can change that like UTF-8 / ISO-8859-1

- action attribute specifies where to submit the form data

- method attribute specifies how to send the form data get or post

        get (default value) - form data is sent as URL variables (Don't use this for sensitive data), limited to 3000 characters

        post - form data sent HTTP post transaction (data not shown as in URL), has no sending limitation

- enctype specifies how the data should be encoded when submittign (works only if method="post") [ application/x-www-form-urlencoded / multipart/form-data / text/plain ]

- target attribute specifies where to display the response received after the server [ _self (default) / _blank / _parent / _top ]

- here is an example with all these attributes


Received response will be displayed in a new page

    <form action="myPhP.php" method="post" enctype="multipart/form-data" accept-charset="UTF-8" target="_blank">

         Received response will be displayed in a new page
    </form>

Input restriction attributes readonly / disabled / required / size / maxlength / max / min / step / pattern

- readonly attribute sets the field olny readable (user cannot change it's value)

- disabled attribute disables the field value WON'T BE SUBMITTED

- requred attribute requires the field to be filled, otherwise the whole form won't be submitted!

- size specifies the field's length in (characters) (only for text based input types, Doesn't work for numbers)

- maxlength specifies the maximum number of characters in a field

- min and max these are very powerful attributes they can have many types of restrictions of maximum and minimum values like numbers and sliders and many more!

- step specifies the legal number intervals (awesome to use like sliders on number based input types)

- pattern this thing can restrict an input scheme with a regExp (very powerful)

Submit attributes form / formaction / formenctype / formmethod / formnovalidate / formtarget

- It is possible for submit buttons to override the form attributes when submiting data

- form refers to the id of the parent's form in case your input field is outside the form element

- formaction replaces the action attribute of the form element, (address where to submit the form )

- formenctype replaces the enctype attribute of the form element, (how to encode input if method="post" or formmethod="post")

- formmethod replaces the method attribute of the form element, (specifies sending method get or post)

- formnovalidate replaces the novalidate attribute of the form element, (submitting the form without validation)

- formtarget replaces the target attribute of the form element, (where to display response after submitting the form)

SYNTAX:     <textarea>                              // specifies a multiline text input (resizeble by default)
    
    // textarea element attributes - - - - - 
            autofocus (boolean)                     //  element gets focus when page loads 
            cols="nr"                               // visible width of the textarea (in average character size)
            dirname="inputName.dir"                 //  submites the text direction of the textarea 
            disabled   (boolean)                    // disables the input 
            form="formId"                           //  specifies which form(s) the textarea belongs to (this is how it is submitted)   
            maxlength="nr"                          // maximum number of characters 
            name="string"                           // name of the textarea (key/property of the element object)
            placeholder="string"                    //  displays a text (hint) in the textarea when empty 
            readonly    (boolean)                   // sets the field read only 
            required    (boolean)                   //  required to fill in order to submit the form 
            rows="nr"                               // visible text lines of the text area
            wrap="soft"  ((default)no new-line character added)   // insert or not a "new line" character at the end of every text row when input submitted to the server 
                 "hard"  (at specified "cols" width a new line character is added)
DEMO

        <input type="submit">
    </form>
    <textarea cols="50" rows="4" name="testTxtArea" dirname="testTxtArea.dir" maxlength="500" placeholder="text area..." form="textIt" warp="hard"></textarea>    
                                                    // (warp="hard") after every 50 characters (because of "cols='50'") a "new line" chacarter is inserted when the browser submits this textarea
                                                    // (fomr="textIt") this input is submitted with the "textIt" form
