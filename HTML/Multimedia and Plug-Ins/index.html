WebVTT - track files .vtt (Video Ttext Track)

- .vtt is a file format for displaying timed text tracks, on the web it is used to add timed text overlays to a <video> element (encoded in UTF-8 format, MIME type text/vtt)

- .vtt files are not supported locally (except Firefox 66.0.2) [TESTED 2019-04-02]

- vtt styling is poorly supported for now, the ::cue selector's (selector) part is partially supported by Chrome only, cue ID's cannot be target, tags in vtt file cannot be targeted, the STYLE within the vtt file is not working, this is maybe the browsers did not implementend these features yet [TESTED 2019-04-02]

            WEBVTT headerName                                 // after the "WEBVTT " everything is optional

      \     text to display                 
                                                              // CUEs and comments are separated with white lines

Plug-In(s) <object> / <embed>

- Plug-Ins are additional softwares instaled in the browser in order to run content what the browser cannot run natively (like: Adobe Flash Player, Java Runtime Environment, ActiveX)

- Plug-Ins are not popular in these days (especially on modile browsers) because HTML5 supports these functionalities natively (video, audio, image, canvas, svg, ect..) so the user does not have to bother to download these additional craps in order tho run the enbeded content

- these embeded external contents might have security concent, this is another important reason to use them as little as possible

- browsers do NOT like Plug-Ins already when I'm testing this Flash Player is already bugging [TESTED 2019-04-04] 

SYNTAX :    <object data="data">                              // embeds external content in the document what can be run by Plug-Ins (can embed multimedia, external content, other HTML page) (does the same as the <embed> tag with some structural differences)   
                <param>                                       // defines parameters for the <object> element 
                fallback message
            </object>
            <embed src="data">                                //  embeds external content in the document what can be run by Plug-Ins (can embed multimedia, external content, other HTML page) (does the same as the <object> tag with some structural differences)   
            
    // <object> element attributes  - - - - - 
        data="URL"                                            // object source
        form="formId"                                         //  specifies the form(s) the object belongs to
        height="nrPx"                                         // height of the object
        name="text"                                           // name of the object
        type="mediaType(MIME)"                                // media type of the embeded content 
        typemustmatch (boolean)                               //  forces to match the type attribute to the actual embeded content type   
        usemap="#mapName"                                     // references a <map> element (client-side image map)    
        width="nrPx"                                          // width of the object      
    
    // <param> element attributes  - - - - - 
        name="text"                                           // parameter name 
        value="text"                                          // parameter value 
    
    // <embed> element attributes  - - - - - 
        height="nrPx"                                         // height of the element 
        src="URL"                                             // ebeded content's source
        type="mediaType(MIME)"                                // media type of the embeded content 
        width="nrPx"                                          // width of the element   
    
// <object> / <param> ------------------------------------------------------------- 
    <object data="flashPlayerData.swf" type="application/x-shockwave-flash" typemustmatch width="500" height="350">       // Flash player Plug-In required to run this type of content (we could also embed any type of content what is supported by an instaled Plug-In, video, audio, image, svg, html document)    
        <param name="autopaly" value="true">                  
        <param name="volume" value="45%">                     // parameters set 
        <param name="fullscreen" value="false">
        <p> Problem Occured </p>                              // fallback message in case the object is not supported or it cannot run the embeded content 
    </object>
        
// <embed> ------------------------------------------------------------------------ 
    <embed src="flashPlayerData.swf" type="application/x-shockwave-flash" width="500" height="350">                       // -! the <embed> does the same but it offers less tools
