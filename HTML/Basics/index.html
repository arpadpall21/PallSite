Description and Demonstration :

- an HTML document is parsed by the browser, unlike XML in HTML elements are "pre-defined" in order for the broswer to "understand" and render them in HTML5 unrecognized elements are rendered as inline elements 

SYNTAX :    <!DOCTYPE html>                                                             <-- not an actual HTML tag, it gives information to the browser of the HTML version (HTML5)  
            <html>                                                                      <-- embeds the document itself
                <head>                                                                  <-- contains information about the document (metadata), not displayed by the browser and it is optional in HTML5
                    <title>text</title>                                                 <-- defines a title of the document (required)
                <head>
                <body>                                                                  <-- visible part of the document
              // elements --------------------------------------------------------------
                <tag attr="value"> text </tag>                                          <-- unlike XML, HTML elements are pre-defined, they can have text content, attributes and other elements 
                <tag>                                                                   <-- closing tags are NOT required they are called empty element
                <Tag> text </Tag>                                                       <-- elements are NOT case sensitive (<tag></tag> is same as <Tag></Tag>)
              // attributes ------------------------------------------------------------
                <tag attr="value"> text </tag>                                          <-- attributes come in the start tag (booleans are allowed)
                <tag attr="value"> text </tag>                                          <-- single or double quotes around attribute values are optional but recommended
              // comments --------------------------------------------------------------
                <!-- comment -->                                                        <-- comments are NOT parsed by the browser 
                </body>
            </html>
    
SYNTAX :    <!DOCTYPE html>                                                            // not an actual html tag, it is a data to the browser about the document version  
            <html>                                                                     // root element of the document
            <head>                                                                     // container for the head elements of the document (metadata container)
            <body>                                                                     // defines the document's body (visible part of the webpage)
            <!-- text -->                                                              // defines a comment (content is not visible and not parsed by the browser)
            
  // html attributes ------------------------------------------------------------------ 
            xmlns="nameSpaceURI"                                                       // namespace attribute for XML (if the content need to be conform to XHTML)

            <!-- invisible comment -->                                                 // coments are not visible by the user
        </body>
    </html>

HTML Entities

- HTML entities are 'escape characters' for the HTML parser, we can use them to display forbidden or special characters in the browser

- the < | > | & | ' | " reserverd characters can be displayed in the page by using HTML entities 

SYNTAX :    &entityName;                                                               // entity name  
            &#entityNumber;                                                            // entity number

dataURL

- the dataURL is basically a data embeded in the HTML document, so instead of requesting the data it is already availabe in the document

- where the data is requested (in href / src / url()) we can simply embed the data prefixed with data:...

- Advantage : the browser does not do a separate request for the external file because the data is there already (like small icons, or text files)

- Disadvantage : big files (like image) are often bigger in encoded format / encoding in base64 increases the file size about 33% 

SYNTAX :    data:MIMEtype

                
 Default: text/plain;charset=US-ASCII
