Statement Description
ALTER TABLE table RENAME TO newTable renames the specified table
ALTER TABLE table ADD col dataType adds a new filed (column) to the table
ALTER TABLE table RENAME COLUMN col TO newCol renames the specified column
ALTER TABLE table MODIFY COLUMN col dataType constraint changes the datatype (and constraint) of the specified field (column)
ALTER TABLE table DROP COLUMN col deletes the specifed filed (column)
CREATE [TEMPORARY] TABLE db.table(col dataType constraint, ...) - creates a new table in the current or (in the specifed) database
- (TEMPORARY) creates a temporary table

- a Temporary table is a session table (only the current user can access it and automatically deleted when the session is closed)

- the user can query temporary tables just like any other tables but Temporary tables CANNOT be listed with any listing statements like SHOW TABLES etc... (because they belong to the connection and not to the database) [TESTED: 2020-10-10 MySQL v8.0.21]

- can have the same name as a base table (in such cases the base table is hidden) (don't do this, creates name confusion)

which belong to the connection
CREATE [TEMPORARY] TABLE db.table AS selectStatement - creates the table what is returned by the select statement (cannot set existirng table) (Primary and Foreign key constraints are not copied)
- (TEMPORARY) creates a temporary table

- a Temporary table is a session table (only the current user can access it and automatically deleted when the session is closed)

- the user can query temporary tables just like any other tables but Temporary tables CANNOT be listed with any listing statements like SHOW TABLES etc... (because they belong to the connection and not to the database) [TESTED: 2020-10-10 MySQL v8.0.21]

- can have the same name as a base table (in such cases the base table is hidden) (don't do this, creates name confusion)

which belong to the connection
DESCRIBE|DESC|EXPLAIN table prints the table structure (fileds, datatype, etc...)
DROP [TEMPORARY] TABLE table, ... - deletes the specified table(s)
- (TEMPORARY) deletes only temporary tables
SHOW CREATE TABLE table prints detailed informations about the current table sturcture (including field constraint settings and constraint Ids)
SHOW FULL TABLES IN db prints all tables and the table type (BASE or VIEW) in the specified database
SHOW TABLES prints all tables in the current database
TRUNCATE TABLE table deletes the table content without deleting the table structure (the table itself)

Statement Description
SELECT *|col, ... INTO OUTFILE 'filePath' options

[FILEDS [TERMINATED BY 'char'] [ENCLOSED BY char]] - specifies the characters how fields are terminated (character between fields) or enclosed

[LINES [TERMINATED BY 'char'] [STARTING BY 'char']] - specifies the characters how lines (records) start and end


- at least one sub-option must be specified

- option parameters can be ASCII characters only

FROM table 		saves the selected table into an external file

- check the secure_file_priv system variable to see where the file can be saved, on Linux (Ubuntu) the default location is '/var/lib/mysql-files/' directory

- the saved file is owned by the mysql user and group and has -rw-r----- privileges

(cannot overwrite existing files)
selectStatement INTO OUTFILE 'filePath' options

[FILEDS [TERMINATED BY 'char'] [ENCLOSED BY char]] - specifies the characters how fields are terminated (character between fields) or enclosed

[LINES [TERMINATED BY 'char'] [STARTING BY 'char']] - specifies the characters how lines (records) start and end


- at least one sub-option must be specified

- option parameters can be ASCII characters only

- this syntax is supported form MySQL v8.0.20 (does the same as above)
- (options) formats the file
LOAD DATA INFILE 'filePath' INTO TABLE db.table options

[FILEDS [TERMINATED BY 'char'] [ENCLOSED BY char]] - specifies the characters how fields are terminated (character between fields) or enclosed

[LINES [TERMINATED BY 'char'] [STARTING BY 'char']] - specifies the characters how lines (records) start and end


- at least one sub-option must be specified

- option parameters can be ASCII characters only

- loads the file content into the specified table

- the target table must have the same number of columns and datatype as the source file has (some datatypes might be parsed tough)

- this statement never override existing records (add duplicates instead)

- the read file location is restricted by the secure_file_priv server variable

- check the secure_file_priv system variable to see from which directory the files can be read, on Linux (Ubuntu) the default location is '/var/lib/mysql-files/' directory


- (options) defines the format how the parser interpretes the file

Constraints
Constraint Description
PRIMARY KEY
uniquely identifies each record in the table (combination of UNIQUE and NOT NULL constraints) (only one PRIMARY KEY allowed in a table)
CREATE TABLE table(col dataType PRIMARY KEY, ...) primary key (or composite key) created when the table is created
CREATE TABLE table(col dataType, ..., PRIMARY KEY(col, ...))
CREATE TABLE table(col dataType, ..., CONSTRAINT ConstId PRIMARY KEY (col, ...))
ALTER TABLE table ADD PRIMARY KEY (col, ...) sets a primary key (or composite key) to an existing table
ALTER TABLE table ADD CONSTRAINT constId PRIMARY KEY (col, ...)
ALTER TABLE table DROP PRIMARY KEY removes the primary key (or composite key) from the table
FOREIGN KEY
- fields refer to existing

- any foreign key value must refer to an existing primary key value

- in other words we cannot insert a foreign key value if the parent primary key field does not contain the value what we want to insert

primary key fields (in the same or another table) (links two tables together)
- FOREIGN KEYs are indexed (INDEX constraint is set with the FOREIGN KEY)
CREATE TABLE table(col dataType, ..., FOREIGN KEY (col) REFERENCES childTable(primKey) [ON DELETE|ON UPDATE const]

Option controls what happens to the FOREIGN KEY value when the linked PRIMARY KEY value is deleted (ON DELETE) or modified (ON UPDATED)

Constraint Values :

- NO ACTION or RESTRICT - the linked PRIMARY KEY value cannot be deleted|updated (DEFAULT: when not set)

- SET NULL - when the linked PRIMARY KEY value is deleted|updated the FOREIGN KEY value will be set to NULL

- CASCADE - when the linked PRIMARY KEY value is deleted|updated the FOREIGN KEY value (record) will be deleted|updated along with the PRIMARY KEY record

- these options can be seen when printing SHOW CREATE TABLE

NOT NULL
enforces the field to always contain a value (cannot hold NULL value)
CREATE TABLE table(col dataType NOT NULL, ...) NOT NULL constraint added when creating a new table
ALTER TABLE table MODIFY COLUMN col dataType NOT NULL adds a NOT NULL constraint to an existing table's column simply by modifying the column
ALTER TABLE table MODIFY COLUMN col dataType simply omit this constraint when modifying the column in order to remove it
UNIQUE INDEX
ensures that all values in the column are different
CREATE TABLE table(col dataType UNIQUE, ...) UNIQUE constraint added when the table is created (constId automatically assigned if created without constraint Id, SHOW CREATE TABLE table to check)
CREATE TABLE table(col dataType, UNIQUE(col, ...))
CREATE TABLE table(col dataType, CONSTRAINT constId UNIQUE (col, ...))
ALTER TABLE table MODIFY COLUMN col dataType UNIQUE adds a UNIQUE constraint to an existing table's column (constId automatically assigned if created without constraint Id, SHOW CREATE TABLE table to check)
ALTER TABLE table ADD UNIQUE(col, ...)
ALTER TABLE table ADD CONSTRAINT constId UNIQUE(col, ...)
ALTER TABLE table DROP CONSTRAINT constId removes referenced UNIQUE constraints (constId) (SHOW CREATE TABLE table to check)
ALTER TABLE table DROP INDEX constId
DEFAULT
specifes a default value for the column
CREATE TABLE table(col dataType DEFAULT defVal, ...) Default constraint set when the table is created
ALTER TABLE table ALTER col SET DEFAULT defVal sets a default constraint of an existing table column
ALTER TABLE table ALTER col DROP DEFAULT drops the default constraint of the specified table column

AUTO_INCREMENT
automatically increments the next value in the column (relative to the previous value) (works only for PRIMARY KEY columns)
CREATE TABLE table(col dataType PRIMARY KEY AUTO_INCREMENT, ...) sets the auto increment constraint when creating a new table
ALTER TABLE table MODIFY COLUMN col dataType AUTO_INCREMENT adds a AUTO_INCREMENT constraint to an existing table's column simply by modifying the column
ALTER TABLE table MODIFY COLUMN col dataType simply omit this constraint when modifying the column in order to remove it
INDEX
- creates an index (indexed data is retrieved more quickly but updated more slowly)
-there are unique and non-unique indexes (unique indexes ensure that all values in the column are different (like the UNIQUE constraint))
CREATE INDEX indexId ON table(col, ...) adds non-unique or unique index(es) to the specified column(s)

ALTER TABLE table DROP INDEX indexId removes the referenced index

DEMO 
// CREATE TABLE -----------------------------------------------------------------------
    CREATE TABLE cars(car_id INT PRIMARY KEY, mark varchar(20), type varchar(20));     // creates a new table with 3 fields   
    CREATE TABLE cars_2 AS SELECT * FROM cars;                                         // creates a new table by copying everithing from the 'cars' table   
    CREATE TABLE cars_3 AS SELECT mark, type FROM cars;                                // creates a new table by copying 2 explicit fileds from the 'cars' table   
    
// ALTER TABLE ------------------------------------------------------------------------
    ALTER TABLE cars RENAME TO kocsik;                                                 // renames the table   
    ALTER TABLE kocsik ADD engine_size INT;                                            // adds a new filed to the 'kocsik' table 
    ALTER TABLE kocsik RENAME COLUMN engine_size TO engine;                            // renames the 'engine_size' filed to 'engine'
    ALTER TABLE kocsik MODIFY COLUMN type INT UNIQUE;                                  // modifies the column datatype and constraint 
    ALTER TABLE kocsik DROP COLUMN type;                                               // deletes the 'type' column (with all stored data in the column)    
    
// TRUNCATE TABLE / DROP TABLE --------------------------------------------------------
    TRUNCATE TABLE kocsik;                                                             // removes all the table's content (but keeps the table structure itself)    
    
    DROP TABLE kocsik;                                                                 // deletes the table from the database 
    
// SHOW TABLES / SHOW FULL TABLES IN / DESC / -----------------------------------------
    SHOW TABLES;                                                                       // prints all tables in the current database 
    SHOW FULL TABLES In myDB;                                                          // prints all tables and table type in the 'myDB' database 
    
    DESC kocsik;                                                                       // prints the table structure of the 'kocsik' table 
    
// SHOW CREATE TABLE ------------------------------------------------------------------

- Temporary Tables 

    mysql> CREATE TEMPORARY TABLE tempTable(col1 INT);                                 // creates the 'tempTable' temporary table (automatically deleted when the session ends)    
    
    mysql> INSERT INTO tempTable VALUES(21);                                           // quering temporary tables work the same way as base tables   
    
    mysql> SHOW TABLES;                                                                // temporary tables are not listed here (or in any table listing commands) (they belong to the connection and not to the database)    
    
// same temporary table name ----------------------------------------------------------
    mysql> CREATE TEMPORARY TABLE Employee AS SELECT * FROM Employee;                  // temporary tables can have the same name as base tables    // -! bad practice (creates name confusion)
                                                                                         // the created 'Employee' temporary table will hide the base 'Emplyee' table    
                                                                                         // from here we modify the 'Emplyee' temporary table (but not the 'Emplyee' base table)
                                                                                         
  // delete temporary table -----------------------------------------------------------
    mysql> DROP TABLE Employee;                                                        // drops the 'Employee' temporary table but not the base 'Employee' table  
    mysql> DROP TABLE Employee;                                                        // -! calling again will drop the base 'Employee' table (this statement drops both temporary and base tables)
    
    mysql> DROP TEMPORARY TABLE Employee;                                              // drops the 'Employee' temporary table only (does not drop base tables)

Constraints

- Constraints specify rules for data in the table, there are column-level and table-level constraints

- if a constraint is violated when inserting new data the insertion is aborted

- implementing constraints it's NOT consistent, there is no clear syntax how we should use them, so I tested individually each of them [TESTED: 2020-08-05]
