NULL (v 8.0.0)
Data Type Description
NULL - no value (not the same as 0 or '' (empty string))
- IS NULL tests for NULL values (in the WHERE clause) (col=NULL doesn't work)
- IS NOT NULL tests for NOT NULL values (in the WHERE clause) (NOT col=NULL doesn't work)

Strings (v 8.0.0)
Data Type Description
Characters and Bites
CHAR(charLength:nr) - fixed length string (stores the number of characters specified by charLength)
- character length from 0 to 255 (Default: 1)
BINARY(byteLength:nr) - fixed length string (stores the number of bytes specified by byteLength)
- byte length from 0 to 255 (Default: 1)
VARCHAR(maxCharLength:nr) - variable length string (stores only the necessary number of characters +1)
- character length from 0 to 65,535
VARBINARY(maxByteLength:nr) - variable length string (stores only the necessary number of bytes +1)
- byte length from 0 to 65,535
Text
TEXT(byteSize:nr) - string (up to 65,535 bytes)
- the (byteSize:nr) parameter can define the right TINYTEXT | TEXT | MEDIUMTEXT | LONGTEXT datatype
TINYTEXT small string (up to 255 characters )
MEDIUMTEXT medium string (up to 16,777,215 characters )
LONGTEXT large string (up to 4,294,967,295 characters )
BLOB
BLOB(byteSize:nr) - BLOB (Binary Large OBject) up to 65,535 bytes
-the (byteSize:nr) parameter can define the right TINYBLOB | BLOB | MEDIUMBLOB | LONGTBLOB datatype
TINYBLOB small BLOB (Binary Large OBject) up to 255 bytes
MEDIUMBLOB medium BLOB (Binary Large OBject) up to 16,777,215 bytes
LONGBLOB medium BLOB (Binary Large OBject) up to 4,294,967,295 bytes
Enum and Set
ENUM(val:str, ...) - must be one out of the specified possible values (strings) or NULL (enumeration storing)
- internally bounds each value to an index, the column holds indexes which reference the bound data (max 65535 indexes)
- storage efficien when predefined data is stored
SET(val:str, ...) - can have zero (NULL) or more values from the passed possibilities (max 64 possible values)
- the order is maintained

Example:

- column col_1 has a set of SET('a', 'b', 'c')

- if we insert a new data with INSERT INTO someTable(col_1) VALUES('b,c,a')

- a data in the column is inserted as a,b,c

when setting multiple values

Numbers (v 8.0.0)
Data Type Description
Bit
BIT(bitSize:nr) - bit value (data represented as bits)
- bit length from 1 to 64 (Default: 1 (8 = 1 byte))
- the value can be set as string, number or by using the special b'bitVal' notation

- string converted to binary according to the current character set (ex: a = 1100001)

- number converted to binary (ex: 97 = 1100001)

- the b'bitVal' notation directly sets the underlying binary map (ex: b'01100001' = 1100001)

Boolean
BOOL|BOOLEAN boolean (0 = false) (1 = true) (same as TINYINT(1))
Integer
TINYINT(displayWidth:nr) [UNSIGNED] [ZEROFILL] - very small integer (exact value without decimals)
- signed range -128 to 127 (Default)

- by Default accepts signed values

- in order to switch to unsigned the UNSIGNED keyword must be specified after the datatype

Example:

ALTER TABLE myTable ADD col_1 TINYINT UNSIGNED;


- unsigned range 0 to 255
- (displayWidth:nr) number of digits to display (Default: 3) does not have any effect on how the data is stored, (works with ZEROFILL option only)
- (UNSIGNED) unsigned range
- (ZEROFILL) zero left pads missing digits (also enables UNSIGNED)

- (ZEROFILL) zero left pads missing digits (also enables UNSIGNED)

- (UNSIGNED) unsigned range

Floating Point
FLOAT(storageSize:nr) [UNSIGNED] [ZEROFILL] - floating point number
- storageSize:nr = 0 - 24 = single precision floating point (Default)
- storageSize:nr = 25 - 53 = double precision floating point
FLOAT(displayTotalDigits:nr, dec:nr) [UNSIGNED] [ZEROFILL] - single precision floating point number (accurate about 7 significant digits but unaccurate decimals)
- (displayTotalDigits:nr) total number of digits to display (including .(period)) (Default: 12), does not have any effect on how the data is stored, (works with ZEROFILL option only)
- (dec:nr) number of decimals to display (Default: 4) (works with ZEROFILL option only)
- (UNSIGNED) unsigned
- (ZEROFILL) zero left pads missing digits (also enables UNSIGNED)
[Deprecated Syntax from MySQL v.8.0.17]
DOUBLE|DOUBLE PRECISION(displayTotalDigits:nr, dec:nr) [UNSIGNED] [ZEROFILL] - double precision floating point number (accurate about 16 significant digits but unaccurate decimals)
- (displayTotalDigits:nr) total number of digits to display (including .(period)) (Default: 22), does not have any effect on how the data is stored, (works with ZEROFILL option only)
- (dec:nr) number of decimals to display (works with ZEROFILL option only) [Deprecated option from MySQL v.8.0.17]
- (UNSIGNED) unsigned
- (ZEROFILL) zero left pads missing digits (also enables UNSIGNED)
Exact Fixed Point
DEC|DECIMAL(totalDigits:nr, dec:nr) [UNSIGNED] [ZEROFILL] - exact fixed point number (significat digits and decimals are 100% accurate)
- (totalDigts:nr) number of total digits including decimals (Default: 10) (max 65)
- (dec:nr) decimals out of the total number of digits (Default: 0) (max 30)
- (UNSIGNED) unsigned
- (ZEROFILL) zero left pads missing digits (also enables UNSIGNED)

Date and Time (v 8.0.0)
Data Type Description
DATE date without time (Syntax: 'YYYY-MM-DD'') (supported range '1000-01-01' to '9999-12-31')
DATETIME - date with time (Syntax: 'YYYY-MM-DD hh:mm:ss')
- supported range '1000-01-01 00:00:00' to '9999-12-31 23:59:59'
- adding DEFAULT CURRENT_TIMESTAMP to the column definition automatically adds the current local date and time as default value to the column
TIMESTAMP - date with time (Syntax: 'YYYY-MM-DD hh:mm:ss')
- visually does the same as DATETIME but the data is stored in seconds since UNIX Epoch (1970-01-01 UTC)
- supported range '1970-01-01 00:00:01'UTC to '2038-01-09 03:14:07'UTC
- adding DEFAULT CURRENT_TIMESTAMP to the column definition automatically adds the current local date and time as default value to the column
TIME - time (Syntax: 'hh:mm:ss')
- supported range '-838:59:59' to '838:59:59'
YEAR - year (Syntax: 'YYYY')
- supported range '1901' to '2155'
