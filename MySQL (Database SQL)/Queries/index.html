Views
Statement Description
CREATE [OR REPLACE] VIEW view AS selectStatement - creates a view

- presotred SQL query which shows up and acts as a table in the database (with some restrictions to the underlying data)

from what is returned by the select statement, view limitations

- cannot contain any variables (user-defined, local or system variables)

- cannot reference TEMPORARY tables

- triggers cannot be attached to a view


- a view is updatable (can modify the underlying data) if

A view is not updatable if:

- contains any joins

- contains any aggregations (COUNT(), AVG(), etc...)

- contains DISTINCT

- contains GROUP BY

- contains HAVING

- contains UNION or UNION ALL

its rows have one-to-one relation to the underlying table(s)
- (OR REPLACE) creates a new or replaces an existing view
SHOW CREATE VIEW view prints detailed informations about the view structure
SHOW FULL TABLES IN db returns all tables in the specified database including views
DROP VIEW [IF EXISTS] view - deletes the specified view from the database
- (IF EXISTS) does not throw an error if the view does not exist

Wildcards
Wildcard Description
SELECT colName, ... FROM tableName WHERE colName NOT LIKE val - the NOT LIKE keyword basically does the same as the = (equal sign) or != (unequal sign) but allows the use of wildcards in values
- wildcards can be used for any data types (ex: '12%' = matches any string, number or date data type starting with 12)
% matches zero or more characters (any character)
_ matches one character (any character)
\char escapes the next character (ex: '\%%' = matches any data which starts with %)

  // aliases -------------------------------------------------
    mysql> SELECT col1 AS c1, col2 AS c2 FROM table_1;                                 // in the returned table 'col1' column is named 'c1' and 'col2' column is named as 'c2'
    
    mysql> SELECT table_1.col1, table_2.col1 FROM table_1, table_2                     // columns from multiple tables

  // WHERE condition -----------------------------------------
    mysql> SELECT * FROM table_1 WHERE col2 IS NULL;                                   // return all records where 'col2' has NULL values  
  
  // HAVING condition ----------------------------------------
    mysql> SELECT * FROM table_1 HAVING col2 IS NULL;                                  // return all records where 'col2' has NULL values   
                                                                                         // -! visually does the same as above but the HAVING clause runs after the SELECT statement   
    mysql> SELECT * FROM table_1 WHERE col1 > 10 HAVING col2 IS NOT NULL;              // there are 2 filters in this statement, the 'WHERE' filters the SELECT statement and the 'HAVING' filters the returned result table    
                                                                                         // returns all records where 'col1' have higher value than 10 then filters again 'col2' cannot have NULL values    
    
  // order ---------------------------------------------------
    mysql> SELECT * FROM table_1 ORDER BY col1;                                        // the returned talbe's records are ordered (ascending order) according to 'col1' column   
    mysql> SELECT * FROM table_1 ORDER BY col1, col2;                                    // ordered first by 'col1' then by 'col2' 
    
  // limit ---------------------------------------------------
    mysql> SELECT * FROM table_1 LIMIT 4;                                              // the returned table has only 4 records

Joins JOIN / LEFT JOIN / RIGHT JOIN / (self join)

- 2 or more tables can be joined together based on a related column between them (a column containing the same data in both tables)

- the related colums don't necessarily have to be in PRIMARY KEY - FOREIGN KEY relationship (but it's common to use joins this way)

- in MySQL there's (INNER) JOIN, LEFT JOIN and RIGHT JOIN but there's no OUTER JOIN

SYNTAX:     SELECT *|table.col, ... FROM leftTable

                
 - here we can specify all the columns from the final joined table what we want to return 

                
 Ex : 

                
 -> SELECT myTable1.col_1, myTable2.col_1 ... = the returned result will show 'col_1' from 'myTable1' and 'col_1' from 'myTable2' 

            
 JOIN rightTable ON leftTable.col=rightTable.col

                
 - columns in both tables on which the join is based 

                
 leftTable.col = column from the left (source) table 

                
 rightTable.col = column from the right (target) table 

            
 moreJoins...

                
 - additional joins (any join type JOIN | LEFT JOIN | RIGHT JOIN) (syntax after the JOIN keyword) 

                
 - multiple joins are chained from left to right 

                
 Example: 

                
 - we want to join 4 tables in this order table_1 -> table_2 -> table_3 -> table_4 

                
 - first table_1 and table_2 are joined and returned 

                
 - then the returned table is joined to table_3 and returned 

                
 - then finally the returned table is joined to table_4 and the final joined table is returned

Views

- views are basically virtual tables showing the underlying data (in other words it's an abstraction over SQL tables), they belong to a database

- we can SELECT query a view just like any other tables, the view always shows the current underlying data (modifying the underlying data will show up in the view immediately) 

SYNTAX:     CREATE [OR REPLACE] VIEW view AS selectStatement                           // creates a view

                
 - presotred SQL query which shows up and acts as a table in the database (with some restrictions to the underlying data) 

            
 from what is returned by the select statement, view limitations

                
 - cannot contain any variables (user-defined, local or system variables) 

                
 - cannot reference TEMPORARY tables 

                
 - triggers cannot be attached to a view 

            
    
                                                                                          a view is updatable (can modify the underlying data) if

                
 A view is not updatable if:

                
 - contains any joins 

                
 - contains any aggregations (COUNT(), AVG(), etc...) 

                
 - contains DISTINCT 

                
 - contains GROUP BY 

                
 - contains HAVING 

                
 - contains UNION or UNION ALL 

            
 its rows have one-to-one relation to the underlying table(s)    
                                                                                          (OR REPLACE) creates a new or replaces an existing view   
            
            SHOW CREATE VIEW view                                                      // prints detailed informations about the view structure     
            
            SHOW FULL TABLES IN db                                                     // returns all tables in the specified database including views   
            
            DROP VIEW [IF EXISTS] view                                                 // deletes the specified view from the database  
                                                                                          (IF EXISTS) does not throw an error if the view does not exist
DEMO 
    mysql> CREATE OR REPLACE VIEW myView AS                                            // creates new or replaces an existing view
        -> SELECT * FROM myTable;                                                        // this is an updatable view because the view references the underlying 'myTable' table's row one-by-one    
    
    mysql> CREATE OR REPLACE VIEW myView2 AS   
        ->   SELECT col1, col3, col1 + col3 AS sum FROM myTable;                       // this view shows only certain columns and does some operations from the underlying table

                                                                                         // I think a view should be used only to see the underlying data and not modify it (modifying data through views can mess things up)

Wildcards

- wildcards can be used in values with the LIKE keyword

- in MySQL there are no character-list style wildcards (ex: [char...] [^char...] [char-char]) unfortunately! 

SYNTAX:     SELECT colName, ... FROM tableName WHERE colName NOT LIKE val              // the NOT LIKE keyword basically does the same as the = (equal sign) or != (unequal sign) but allows the use of wildcards in values    
                                                                                          wildcards can be used for any data types (ex: '12%' = matches any string, number or date data type starting with 12)    
            
            %                                                                          // matches zero or more characters (any character)
            _                                                                          // matches one character (any character)
            
            \char                                                                      // (ex: '\%%' = matches any data which starts with %)
DEMO 
    mysql> SELECT * FROM myTable WHERE col_1 = 'a%';                                   // -! the 'a%' is not treated as wildcard because the equal sign is used instead of the LIKE keyword   
    
    mysql> SELECT * FROM myTable WHERE col_1 LIKE 'a%';                                // matches any values in the 'col_1' field which starts with 'a'   
    
    mysql> SELECT * FROM myTable WHERE col_1 LIKE 'a_';                                // matches any values in the 'col_1' field which starts with 'a' and followed by one character    
    
    mysql> SELECT * FROM myTable WHERE col_2 LIKE '%00';                               // matches any values in the 'col_2' field which ends with '00'           // -! works even if the 'col_2' field contains number data types    
    
    mysql> SELECT * FROM myTable WHERE col_3 LIKE '2020-01-%';                         // matches any values in the 'col_2' field which starts with '2020-01-'   // -! works even if the 'col_3' field contains date data types    
    
    mysql> SELECT * FROM myTable WHERE col_1 LIKE '\%%'                                // matches any values in the 'col_1' field which starts with '%'  
    
    mysql> SELECT * FROM myTable WHERE col_1 LIKE '%\_'                                // matches any values in the 'col_1' field which ends with '_'    
    
// NOT LIKE ---------------------------------------------------------------------------
    mysql> SELECT * FROM myTable WHERE col_1 NOT LIKE 'a%'                             // returns all values in 'col_1' which does not start with 'a'  
    
    mysql> SELECT * FROM myTable WHERE col_1 NOT LIKE '%x%'                            // returns all values in 'col_1' which does not contain any 'x' characters
