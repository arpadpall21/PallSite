Description and Demonstration:

- JavaScript data is dynamic (The same variable can be used to store different data)

- JavaScript has 7 Primitive Data types, any other data is considered as complex data (object)

Primitive Data

- a primitive data is also constructed with the corresponding Object Constructor, they themselves do not have any properties or methods but they can still use their Constructor's properties and methods

- in JavaScript primitives are immutable (we cannot have multiple references to the same primitive (like objects can)) and executes faster than objects

Primitives in JavaScript [Ordered: Alphabetically]
Data Description
string - represents a textual data

boolean - represents a logical data with two possible value true or false

DEMO 
// undefined VS null ------------------------------------------------------------------
    var x;                                                                             // -> undefined              // undefined = data without value  
    var x = null;                                                                      // -> null                   // null = no data at all   
    
// primitives are immutable  ----------------------------------------------------------
    var str1 = "myString"; 
    var str2 = str1;                                                                   // primitive data is immutable (str2 variable has a copy of str1 variable's primitive)   
    
    str1;                                                                              // -> "myString";
    str2;                                                                              // -> "myString";
    
    str1 = "newString";
    
    str1;                                                                              // -> "newString";
    str2;                                                                              // -> "myString";            // each variable has its own primitive

Data Type checking and Conversion typeof / valueOf()

- if we operate on a "wrong" data type JavaScript will try to convert the value to a "right" data type, this can produce unexpected results sometimes

- we can convert a data type by using Constructors (ex: String()) or various methods, Careful! some conversion types might be tricky see conversion table above

DEMO 
// primitive data ---------------------------------------------------------------------
    typeof "myString"                                                                  // -> "string"
    typeof 21                                                                          // -> "number"
    typeof true                                                                        // -> "boolean"
    typeof undefined                                                                   // -> "undefined" 
    typeof null                                                                        // -> "object"       // !! this is a bug in JavaScript it should be "null"
    typeof Symbol                                                                      // -> "symbol"       // ES6
    typeof 21n                                                                         // -> "bigint"       // ES11
    
// complex data -----------------------------------------------------------------------
    typeof function myFunction(){}                                                     // -> "function"     // functions are complex datas but returned as "function"
    typeof {}                                                                          // -> "object"       // any other data will be returned as "object"

Symbols Symbol() / Object.getOwnPropertySymbols() / obj[symbol]

- symbols are unique primitive data types in JavaScript, they are created by the Symbol() function, they only can be used as object keys

- symbol are neither private nor secure, they never conflict with each other or with oder object keys because they are unique (in some other programing languages are called "atom")

- through the Symbol function's static properties we can assess to special built-in objects called "Well-Known Symbols" and through static methods we can assess to the Global Symbol Registry

DEMO 
    var mySymbolA = Symbol("sym-a"); 
    var mySymbolB = Symbol("sym-b");  
    
    var myObj = {
        nativeProp:'someValue', 
        [mySymbolA]:"symbolVal-A",
        Symbol('sym-c'):'symbolVal-C'                                                  // symbol set as object key but it isn't referenced  
    };  
    
    myObj[mySymbolB] = "symbolVal-B" 
    
    myObj[mySymbolA];                                                                  // -> 'symbolVal-A'
    myObj[mySymbolB];                                                                  // -> 'symbolVal-B'
    
    mySymbolA = null
    myObj[mySymbolA];                                                                  // -> undefined              // symbol not referenced by 'mySymbolA' variable anymore  
    
    for(let i in myObj){
        i;                                                                             // -> 'nativeProp'           // symbols are not shown on iteration   
    }
    
// Object.getOwnPropertySymbols() -----------------------------------------------------
    myObjSymbols = Object.getOwnPropertySymbols(myObj);                                // -> [Symbol(sym-a), Symbol(sym-c), Symbol(sym-b)]   // object has 3 symbols  
                                                                                       // this how we can access object symbols if they are not referenced by any variables   
    
    myObj[myObjSymbols[0]];                                                            // -> 'symbolVal-A'
    myObj[myObjSymbols[1]];                                                            // -> 'symbolVal-B'
    myObj[myObjSymbols[2]];                                                            // -> 'symbolVal-C'

Global Symbols and the Global Symbol Registry Symbol.for() / Symbol.keyFor()

- the "Global Symbol Registry" in NOT the "window" object, it is a higher level registry in JS which can be used by any realms (such as Iframe, Web Worker ...)

- "Global Symbols" are basically symbols in the Global Symbol Registry, ("Well-known Symbols" are NOT Global Symbols)

DEMO 
    var localSymbol = Symbol('local');                                                 // local symbol 
    var globalSymbol = Symbol.for('global');                                           // set (register) a symbol in the Global Symbol Registry 
    
    Symbol.keyFor(localSymbol);                                                        // -> undefined          // symbol not found in the Global Symbol Registry
    Symbol.keyFor(globalSymbol);                                                       // -> 'global'           // symbol registered in the Global Symbol Registry   
    
// using global symbols ---------------------------------------------------------------
    let myObj_ = {
        [localSymbol]: 'local Symbol Value',
        [globalSymbol]: 'global Symbol Value'}                                         // global symbols do the same as local onces (they're just registered in high-level (GSR))   
        
    myObj_;                                                                            // -> {Symbol(global):"global Symbol Value", Symbol(local):"local Symbol Value"}

Well-Known Symbols

- Well-Known Symbols are special objects, they configure how JavaScript treats an object

- if JavaScript recognizes a Well-Known Symbol as inherited or as native object key it will treat the host object in a certain way

- we can access Well-Known Symbols as static properties of the Symbol function (ex: Symbol.toString)

DEMO 
// check if the object/value implements a symbol --------------------------------------
    [][Symbol.iterator];                                                               // -> values() { [native code] }             // Array implements the iterator symbol 
    ' '[Symbol.iterator];                                                              // -> [Symbol.iterator]() { [native code] }  // string (primitive value) implements the iterator symbol    
    
    {}[Symbol.iterator];                                                               // -> undefined                              // Object does not implement the iterator symbol
    
// [Symbol.toPrimitive] ---------------------------------------------------------------
    var testObj = {p1:1, [Symbol.toPrimitive](hint){
        if(hint == 'number') return 10;                                                // if the object is converted to a number it will return this value 
        if(hint == 'string') return 'f@szagyerek';                                     // if the object is converted to a string it will return this value
        return 'default'                                                               // if the object is called as a primitive value it will return this value 
    }};
    
    Number(testObj);                                                                   // -> 10             // object converted to a number 
    String(testObj);                                                                   // -> 'f@szagyerek'  // object converted to a string 
    '=' + testObj;                                                                     // -> '=default'     // object called as primitive value
