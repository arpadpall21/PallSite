File System Types

Updated ( 2020-01-30 )

MySite> Operating Systems> File System Types

File System Description
Microsoft
FAT12 (File Allocation Table) - (1980)
- max volume size 32MB
- max file size 32MB
FAT16 - (1984)
- max volume size 16GB
- max file size 2GB/4GB
FAT32 - (1996) compatible with other OS
- max volume size 32GB (windows) / 2TB (other OS) / 16TB (theoretical)
- max file size 4GB
NTFS (New Technology File System) - (1993) not compatible with non-Windows OS, macOS read-only
- max file size 16EB (exabytes)
- supports file permission encryption
- supports journaling file system (to avoid corruption)
exFAT (Extended File Allocation Table) - (2006) optimized for high-capacity USB flash drives and memory cards
- max file size 16EB (exabytes)
- supported by Windows, macOS, Android and partially by LINUX
Linux
EXT (Extended File System) - (1992) Linux only
- max partition size 64MB
EXT2 - (1993) Linux only
- max volume Size 32TB
- max file size 2TB
EXT3 - (2001) Linux only
- max volume size 32TB
- max file size 2TB
- supports journaling file system (to avoid corruption)
EXT4 - (2008) Linux only
- max volume size 1EB (exabytes)
- max file size 16TB
- supports journaling file system (to avoid corruption)
Apple
HFS (Hierarchical File System) - (1985) Apple only
- max volume size 2TB - max file size 2GB
HFS+ - (1998) Apple only
- max volume size 8EB
- max file size 8EB
- supports journaling file system (to avoid corruption)
APFS (Apple File System) - (2017) Apple only, optimized for flash and SSDs
- max volume size 8EB
- max file size 8EB
Other
ZFS (the Z File System) - (2006) developed by Sun Microsystems and developed by OpenZFS
- supported on Linux, FreeBSD, trueOS
- max volume size 256YB (yobibytes)
- max file size 16EB
- has physically integrated volume manager to control storage hardware
- has enhanced protection against data corruption
