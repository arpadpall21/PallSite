Compiled VS Interpreted Programming language

Compiled Programming Languages (like C, C++)

- uses a compiler which compiles the script into machine readable code (binary code) first, then this code can be executed easily

- Advantage: faster to execute because the code is already compiled into machine code, has more control over the underlying hardware

- Disadvantage: requires 2 steps to test it, (1 the source code is compiled into machine code) (2 run the program)

Interpreted Programming Languages (like JavaScript, Python)

- uses an interpreter which compiles the script when executed (Juist In Time execution)

- Advantage: when tested no extra step needed, we can execute the script directly, its platform independent (the interpreter can run on different more easily)

- Disadvantage: slower than compiled languages, because the interpreter must compile the script first

Compile VS Runtime

- Compile Time when the script is converted into Machine Code, so the computer understands it

- Run Time is when the computer executes "runs" the compiled Machine Code

Static VS Dynamic Programming Language

Statically Typed Programming Languages (like Java)

- check for syntax and type errors at compile time

- variables must be type declared (which is checked at compile time) before values are assigned to them at run time

Dinamically Typed Programming Languages (like JavaScript)

- only check for syntax errors at compile time

- so variables don't have to be type declared and there's no type checking at compile time

Prototype based VS Class based Programming Language

- in a prototype based programming language an object can be derived from a class or from an object (the object's 'prototype)

- in a class based programming language an object can be derived from a class
